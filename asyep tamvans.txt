kesimpulan:
Git adalah sistem kontrol versi (VCS) terdistribusi yang populer untuk melacak perubahan pada file teks. 
Git banyak digunakan oleh para developer untuk mengelola kode sumber proyek perangkat lunak.

Manfaat Menggunakan Git:
1. Meningkatkan kolaborasi: Git memudahkan tim untuk bekerja sama secara efektif dalam proyek.
2. Meningkatkan kualitas kode: Git membantu Anda mengidentifikasi dan memperbaiki bug lebih cepat.
3. Meningkatkan keamanan: Git memungkinkan Anda untuk melacak dan memulihkan file yang dihapus atau diubah secara tidak sengaja.
4. Meningkatkan efisiensi: Git membantu Anda menghemat waktu dan ruang penyimpanan.

Istilah Penting Git:
1. Repositori: Lokasi di mana semua file proyek dan riwayat perubahannya disimpan.
2. Commit: Snapshot dari keadaan repositori pada waktu tertentu.
3. Branch: Salinan dari repositori yang memungkinkan Anda untuk mengerjakan perubahan tanpa memengaruhi kode utama.
4. Remote: Salinan repositori yang disimpan di server lain.
5. Merge: Menggabungkan perubahan dari satu branch ke branch lain.
